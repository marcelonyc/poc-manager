# Managing POCs

Learn how to effectively manage POC engagements throughout their lifecycle.

## Overview

POC management involves monitoring progress, updating status, managing tasks, and ensuring successful outcomes.

## POC List View

### Accessing POCs

1. Click **POCs** in the main navigation
2. View all POCs you have access to

### POC Cards

Each POC card displays:

- **Title**: POC name
- **Customer**: Company name
- **Date Range**: Start to end date
- **Status**: Current POC state
- **Progress**: Visual progress indicator
- **Quick Actions**: View, Edit buttons

### Filtering and Sorting

!!! info "Coming Soon"
    Advanced filtering and sorting options are planned for future releases.

## POC Detail View

### Opening a POC

Click on any POC card to open the detailed view.

### Navigation Tabs

The POC detail page includes several tabs:

=== "Basic Info"
    View and edit core POC information:
    
    - Title and description
    - Customer company
    - Date range
    - Executive summary
    - Objectives
    - Products

=== "Tasks & Groups"
    Manage POC tasks:
    
    - View all tasks
    - Update task status
    - Add new tasks
    - Organize with task groups
    - Track completion

=== "Success Criteria"
    Define and track success metrics:
    
    - Add criteria
    - Update status (Met/Not Met)
    - Track progress
    - Document outcomes

=== "Resources"
    Manage supporting materials:
    
    - View attached resources
    - Add new resources
    - Organize by task
    - Share with customer

=== "Comments"
    Communication hub:
    
    - Internal comments (team only)
    - External comments (customer visible)
    - Comment history
    - Attachments

## Editing POC Details

### Updating Basic Information

1. Navigate to the **Basic Info** tab
2. Click **Edit** button
3. Modify any field:
   - Title
   - Description
   - Dates
   - Customer info
   - Objectives
4. Click **Save Changes**

### Best Practices for Updates

- Keep information current
- Document significant changes
- Notify team of major updates
- Maintain change history awareness

## Task Management

### Viewing Tasks

The **Tasks & Groups** tab shows:

- All tasks with current status
- Task groups for organization
- Progress indicators
- Action buttons

### Updating Task Status

To change a task's status:

1. Find the task in the list
2. Click the status dropdown
3. Select new status:
   - **Not Started**: Task hasn't begun
   - **In Progress**: Currently working
   - **Completed**: Task finished
   - **Blocked**: Cannot proceed
4. Add a comment explaining the change (optional)

### Adding Tasks During POC

You can add tasks even after POC creation:

1. Click **Add Task** button
2. Choose from template or create custom
3. Fill in task details
4. Assign to team member
5. Save the task

### Removing Tasks

!!! warning "Be Careful"
    Removing tasks deletes associated comments and history.

To remove a task:

1. Find the task
2. Click delete icon
3. Confirm deletion
4. Task is permanently removed

## Progress Tracking

### Progress Indicators

POCs show progress in multiple ways:

- **Overall Progress Bar**: Percentage complete
- **Task Completion Count**: X of Y tasks complete
- **Status Colors**: Visual status indicators
- **Timeline**: Days elapsed vs. total duration

### Calculating Progress

Progress is automatically calculated based on:

- Number of completed tasks
- Task group completion
- Success criteria met
- Overall POC status

### Sharing with Customers

Once a customer is invited:

- They can view POC details
- See tasks and progress
- Add external comments
- Access shared resources

### Managing Customer Feedback

Customer feedback appears as:

- External comments on tasks
- Status updates they provide
- Questions they ask

### Communication Best Practices

- Respond to customer comments promptly
- Keep internal discussions internal
- Be transparent about progress

## Reporting

### Progress Reports

Generate reports on:

- Task completion rates
- Timeline adherence
- Success criteria status
- Resource usage

### Document Generation

Create POC documents:

1. Click **Generate Document** button
2. Choose format (PDF or Markdown)
3. Document is generated with:
   - POC details
   - Tasks and status
   - Success criteria
   - Comments history
4. Download and share

## POC Lifecycle

### Active POCs

While POC is active:

- Update tasks regularly
- Monitor progress
- Respond to comments
- Track against timeline
- Adjust as needed

### Completing POCs

When POC is complete:

1. Ensure all tasks are updated
2. Verify success criteria
3. Generate final report
4. Gather feedback
5. Archive POC

## Best Practices

### Regular Updates

- Update task status daily
- Review progress weekly
- Communicate changes promptly
- Document issues as they arise

### Organization

- Keep tasks well-organized
- Use task groups effectively
- Tag and categorize appropriately

## Common Scenarios

### POC Running Behind Schedule

If falling behind:

1. Identify blocking issues
2. Update task statuses to "Blocked"
3. Add comments explaining delays
4. Discuss timeline adjustment
5. Communicate with customer

### Scope Changes

When scope changes occur:

1. Update POC description
2. Add/remove tasks as needed
3. Adjust success criteria
4. Update timeline if necessary
5. Document the change reason

### Customer Not Responding

If customer is unresponsive:

1. Send reminder via comments
2. Escalate internally
3. Document communication attempts
4. Consider timeline adjustment
5. Discuss with management

## Troubleshooting

### Cannot Update POC

If you can't update the POC:

- Verify you have proper permissions
- Check POC isn't in read-only state
- Ensure you're online
- Try refreshing the page

### Changes Not Saving

If changes won't save:

- Check all required fields
- Verify data format is correct
- Check internet connection
- Look for error messages
- Try saving again

### Customer Cannot See Updates

If customer can't see updates:

- Verify they're invited
- Check their account is active
- Ensure changes were saved
- Confirm they're viewing correct POC
- Have them refresh their page
